## 0. 개요

| 항목 | 값 |
|---|---|
| 총 예문 수 | 1,100문 |
| N5 예문 | 300문 (회화 150 + 신문 50 + 비즈니스 100) |
| N4 예문 | 300문 (회화 150 + 신문 70 + 비즈니스 80) |
| N3 예문 | 500문 (회화 200 + 신문 150 + 비즈니스 150) |
| 활용 기간 | 16주 (전 학습 기간 통합) |
| 하루 노출 | 평균 10문 (낭독 + 청취 + 1문 작문) |
| 문장당 시간 | 2분 (낭독 30초 + 청취 30초 + 의미 확인 30초 + 인출 30초) |

### 학습 원리 (소스 01 연계)
- **이해 가능한 인풋 + 1 (Krashen i+1)**: 학습자 수준보다 약간 위의 문장 노출
- **듀얼 코딩**: 텍스트 + 음성 (NotebookLM Audio Overview)
- **인터리빙**: 회화·신문·비즈니스 3계열을 섞어 변별
- **생성 효과**: 매 10문당 1문은 자기 작문으로 변환
- **수면 응고**: 취침 전 5문 청취 (인풋만, 출력 없이 안정화)

## 1. 예문 분류 체계

| 계열 | 특징 | 목적 |
|---|---|---|
| **회화체** | です·ます 정중체 + 보통체, 단축어, 종조사 (ね·よ·かな·っけ) | 청해·일상회화 대비 |
| **신문체** | だ·である 문어체, 한자어 비율 높음, 무인칭 표현 | 독해 대비 |
| **비즈니스체** | 경어 (존경·겸양·정중), 격식 표현 | 청해·실생활·N2 가교 |

### 표기 규칙
- 모든 문장은 **번호 + 일본어 + (한국어 뜻) + [출현 문법·어휘]** 형식
- 한자에는 첫 등장 시 후리가나 병기
- 회화체는 **A·B** 화자 구분 가능
- 비즈니스체는 시나리오 제목 포함

## 13. 인터리빙 학습 권장 조합

### 13-1. 레벨 인터리빙 (N5·N4·N3 혼합)
- 동일 주제·다른 레벨 비교 (예: 첫 인사 N5 #1 vs N4 #301 vs N3 #951)
- 어휘·문법 복잡도 상승을 직관적으로 체감
- N5→N3 가교 효과 강화

### 13-2. 계열 인터리빙 (회화·신문·비즈니스 혼합)
- 동일 화제를 3계열로 비교 (예: 경제 뉴스 N5 #170 vs N4 #460 vs N3 #810)
- 격식별 어휘·문법 변환 인지
- 실전 청해·독해·작문 적응력 향상

### 13-3. 문법 인터리빙 (동일 문법 다양 예문)

## 15.활용 팁

### 15-1. 예문을 코드 주석처럼 활용
- 각 N3 문법(소스 11) 카드의 예문 필드에 본 코퍼스 번호 링크
- 예: `~ようになる :: #748`
- 학습 진행 중 즉시 클릭하여 실제 사용 문맥 확인

### 15-2. 예문 코퍼스를 검색 가능한 데이터셋으로
- 본 마크다운을 SQLite·grep 등으로 색인
- 어휘·문법으로 즉시 예문 검색 (예: `grep "ざるを得ない"`)
- 시스템 프로그래머 직무 관련 단어(技術·開発·制御·効率化) 집중 학습 가능

### 15-3. 계층적 학습 흐름 (state machine 관점)
```
[N5 회화] → [N5 신문·비즈니스] → 
[N4 회화] → [N4 신문·비즈니스] → 
[N3 회화] → [N3 신문] → [N3 비즈니스]
                ↓ (조건: 정답률 75%↑)
        [N2 가교 (부록-B 활용)]
```

### 15-4. 인터리빙을 단위 테스트로
- 16주차 종합 청취 시 셔플된 1,100문을 단위 테스트 suite로 간주
- 각 문장 = test case
- 한국어 → 일본어 재생성 = assertion
- 정답률 70% 이상 = 합격
